1. Cài đặt & Cấu hình Tools On-Premise: 

[ ] Deploy MinIO: 

Chạy MinIO Server (Docker). 

Truy cập giao diện web, tạo access key và secret key. 

Tạo các bucket với cấu trúc rõ ràng: 

landing-zone: Nơi nhận file upload ban đầu. 

bronze-layer: Lưu dữ liệu RAW (dạng Delta table). 

silver-layer: Lưu dữ liệu đã qua xử lý, làm sạch. 

gold-layer: Lưu dữ liệu đã được tổng hợp, sẵn sàng cho phân tích.


Ingestion - Thu thập dữ liệu: 

[ ] (File Upload): 

Trong Dagster, tạo một Software-defined Asset tên raw_sales_file. 

Trong asset này, viết Python code sử dụng thư viện boto3 để đọc file mới nhất từ bucket landing-zone. 

Viết code để chuyển file CSV/JSON thành định dạng Parquet và lưu vào một bảng Delta/Iceberg trong bucket bronze-layer. 

Tạo một Sensor trong Dagster, đặt tên là new_file_sensor. Sensor này sẽ định kỳ kiểm tra bucket landing-zone bằng boto3, nếu có file mới sẽ trigger job để chạy asset raw_sales_file. 

[ ] (CDC): 

Cấu hình Debezium Connector để theo dõi bảng customers trong database PostgreSQL. 

Cấu hình Debezium để đẩy các change events (dạng JSON) vào một topic trên Kafka (nếu dùng) hoặc trực tiếp vào một thư mục trên MinIO. 

Trong Dagster, tạo một asset tên bronze_cdc_customers để đọc các file JSON này, bóc tách dữ liệu (before, after, op) và ghi vào một bảng Delta/Iceberg tại bronze-layer.
